**삼품관리(33Auto)**

2025.09. ~ 2025.11.

현대오토에버 모빌리티 SW스쿨 웹/앱 2기

https://github.com/33-Auto

**프로젝트 개요**

삼품관리는 **자동차 부품 산업에서 발생하는 구매·생산·재고·판매·인사 관리 업무를 하나의 ERP 시스템으로 통합**하는 프로젝트입니다.

분산되어 있던 기준정보와 업무 흐름을 정리해 데이터 정합성을 확보하고, 조직·권한 단위로 업무를 관리할 수 있는 구조를 목표로 설계되었습니다.

본 프로젝트에서는 ERP 전체 기능을 모바일로 구현한 것이 아니라,

**현업에서 자주 사용되는 핵심 ERP 기능을 모바일 환경에서도 처리할 수 있도록 일부 기능을 모바일 앱으로 확장**하는 데 초점을 맞췄습니다.

이를 통해 현장이나 이동 중에도 주요 정보를 확인·처리할 수 있는 운영 환경을 만드는 것이 목표였습니다.

**담당 업무 및 기여**

- **Android / iOS 모바일 앱 동시 개발 담당**
    
    Compose와 SwiftUI를 사용해 동일한 기능을 두 플랫폼에서 병렬로 구현했습니다.
    
- **아키텍처 및 도메인 로직 통일**
    
    Clean Architecture + MVVM 구조를 기준으로, 플랫폼이 달라도 동일한 비즈니스 규칙과 화면 흐름이 유지되도록 도메인 구조를 맞췄습니다.
    
- **API 연동 및 상태 관리 구조 정리**
    
    네트워크 계층과 화면 상태 처리 방식을 표준화해, 로딩·성공·실패 흐름이 일관되게 동작하도록 설계했습니다.
    
- **프론트엔드 일부 기능 보조 개발**
    
    React 기반 프론트엔드에서 부품 마스터·인사관리 기능 중 일부 화면 및 로직 구현을 지원했습니다.
    

**사용 기술**

- Android - `Compose`, `Retrofit`, `Clean Architecture`, `Hilt`
- iOS - `SwiftUI`, `Alamofire`, `Clean Architecture`
- FrontEnd - `React`, `TailwindCSS`, `Feature-Sliced Design`

**성과**

- **모바일 ERP 핵심 기능의 안정적 확장 -** 기존 ERP 시스템의 주요 기능을 모바일 환경에서도 사용할 수 있도록 구현해 활용 범위를 넓혔습니다.
- **멀티 플랫폼 아키텍처 이해도 향상** - Android와 iOS를 동시에 개발하며, 언어와 프레임워크가 달라도 도메인과 아키텍처를 중심으로 통일하는 경험을 쌓았습니다.
- **Clean Architecture 실전 적용 경험** - 단순 이론 학습이 아닌, 실제 기능 개발 과정에서 계층 분리와 의존성 관리의 중요성을 체감했습니다.

**프로젝트 개요**

중고차를 **일반거래 + 경매**로 사고파는 플랫폼으로, 판매자는 매물을 등록하고 구매자는 검색·필터를 통해 차량을 탐색한 뒤 거래를 진행합니다.

경매는 **실시간 낙찰자/낙찰가/남은 시간**이 즉시 반영되도록 설계해, 참여자 간 데이터 불일치 없이 거래가 성사되도록 구현했습니다.

**담당 업무 및 기여**

- **경매 거래 플로우 전반 구현**
    
    입찰, 실시간 낙찰가/낙찰자 갱신, 거래 성사까지 iOS에서 전체 흐름을 설계·개발했습니다.
    
- **Realtime DB 기반 정합성/동시성 처리**
    
    동시에 입찰이 들어오는 상황에서 낙찰 데이터가 어긋나지 않도록 동시성 체크 로직을 적용했습니다.
    
- **실시간 타이머(서버 시간 기준) 구현**
    
    남은 시간을 초 단위로 표시하고, 서버 시간 기반으로 계산해 클라이언트 편차로 인한 오차를 줄였습니다.
    
- **거래 계약서 자동 생성(PDF)**
    
    거래 성사 시 차량/금액/개인정보를 기반으로 계약서를 PDF로 생성하고, 조회·저장 흐름까지 구현했습니다.
    
- **담당 범위 명확화**
    
    매물 등록/매물 검색 기능을 제외한 핵심 거래 기능을 iOS에서 담당했습니다.
    

**사용 기술**

- iOS - `SwiftUI`, `Alamofire`, `Firebase Realtime Database`, `Google Authorization`

**성과**

- **실시간 경매의 신뢰도 확보** - 낙찰자, 낙찰가, 시간 정보가 실시간으로 일치하는 구조를 구현해 경매 UX를 안정화했습니다.
- **거래 완료까지 원스톱 제공** - 경매 종료 후 계약서 PDF 자동 생성으로 거래 마무리 과정을 앱 내부에서 완결했습니다.

**프로젝트 개요**

InsideMovie는 사용자의 **감정 취향**을 기반으로 영화를 추천하는 플랫폼입니다.

사용자는 선호 영화 선택으로 감정 프로필을 만들고, 메인 화면의 **5가지 감정(인사이드아웃 모티브) 슬라이드**를 조절해 추천 결과를 바꿀 수 있습니다.

리뷰/평점 작성 시 AI가 텍스트를 분석해 감정을 시각화하고, 리뷰 누적에 따라 영화의 감정 프로필도 갱신됩니다. (TMDB 연동: 박스오피스/검색/필터/상세 제공, 주간 영화 투표/관리 기능 포함)

**담당 업무 및 기여**

- **Frontend 리드(관리자 페이지 제외 전 화면)**
    
    메인(감정 슬라이더+추천), 영화 목록/검색·필터, 상세, 리뷰/평점/감정 시각화까지 사용자 플로우 전체를 설계·구현했습니다.
    
- **감정 기반 추천 UX 구현**
    
    감정 슬라이더 변경 → 추천 갱신 흐름을 만들고, 사용자 조작 중 서버 호출이 과해지지 않도록 지연 전달(쓰로틀링) 방식으로 요청 타이밍을 제어했습니다.
    
- **데이터 연동/상태 관리 정리**
    
    TMDB 데이터(박스오피스/검색/상세)와 내부 추천/리뷰 데이터를 함께 다루며 로딩/성공/실패 상태를 화면 단위로 정리해 끊김 없는 화면 전환에 집중했습니다.
    
- **Backend 일부 기여**
    
    로그인/회원가입/사용자 정보 조회 등 사용자 관련 API 일부를 직접 구현하고, FE 관점에서 API 요청/응답 형태를 맞춰 실제 화면 흐름이 매끄럽게 이어지도록 조정했습니다.
    

**사용 기술**

- Backend - `Java`, `Spring Boot`, `JPA`, `Python`, `Flask`
- Frontend - `TypeScript`, `React`, `TailwindCSS`

**성과**

- **감정 슬라이더 기반 추천 경험 완성** - 사용자가 감정 상태를 조절하면 추천 결과가 즉시 바뀌는 메인 UX를 구현해 이 서비스의 한 줄 가치가 첫 화면에서 바로 전달되게 만들었습니다.
- **서버 부하/UX 이탈 포인트 감소** - 슬라이더 조작을 그대로 API 호출로 연결하지 않고, 요청 타이밍을 제어해 불필요한 호출을 줄이면서도 추천 갱신 체감은 유지했습니다.
